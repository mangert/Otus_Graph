## 2. Реализация структур данных

Все представления реализованы в виде шаблонных классов, наследующих от интерфейса `IGraph<Vertex, EdgeInfo, Directed>`.

**Общие особенности реализации:**
- Шаблонный параметр `Directed` определяет тип графа (ориентированный/неориентированный)
- Для ускорения поиска вершин и рёбер используются вспомогательные индексные структуры
- Для представлений, не поддерживающих хранение информации о рёбрах, `EdgeInfo` исключен из параметров шаблона (используется `bool`)
- Везде, где требуется сравнение вершин, используется концепт `Comparable`

---

### 2.1 Перечисление множеств ([GraphEnumeration.h](../headers/graph/GraphEnumeration.h))

**Особенности:**
- Прямая реализация математического определения графа
- Для неориентированных графов рёбра хранятся в обоих направлениях в индексе
---

### 2.2 Матрица смежности ([GraphAdjMatrix.h](../headers/graph/GraphAdjMatrix.h))

**Особенности:**
- `std::optional` позволяет отличать отсутствие ребра от нулевого веса
- Для неориентированных графов матрица симметрична
- При удалении вершины используется перестановка с последней для сохранения компактности
- Дополнительно сохраняем в классе отображение вершины на индекс и обратно (для удобства поиска и возможности хранить информацию о вершине)

---

### 2.3 Матрица инцидентности ([GraphIncMatrix.h](../headers/graph/GraphIncMatrix.h))

**Значения в матрице:**
- Неориентированный граф: 0, 1 (2 для петли)
- Ориентированный граф: -1 (начало), 1 (конец), 2 (петля)

**Особенности:**
- Дополнительно сохраняем отображение вершин и ребер на индексы для удобства поиска
---

### 2.4 Перечень рёбер ([GraphEdgeList.h](../headers/graph/GraphEdgeList.h))

**Особенности:**
- Дополнительно сохраняем множество вершин и индекс ребер
- Для неориентированных графов в индексе хранятся оба направления

---

### 2.5 Векторы смежности ([GraphAdjVectors.h](../headers/graph/GraphAdjVectors.h))

**Особенности:**
- Дополнительно сохраняем актуальную максимамальную степень вершинц
- Свободные ячейки заполнены `std::nullopt` 
- Информация о рёбрах отдельно не сохраняется, ребра определяются через вершины
- Дополнительно сохраняем отображение вершин на индексы (для удобства поиска)
- Реализована дополнительная функция `shrink_to_fit()`, позволяющая убрать "дырки", которые могут появиться в результате удаления ребер

---

### 2.6 Массивы смежности ([GraphAdjArray.h](../headers/graph/GraphAdjArray.h))

**Особенности:**
- Компактное хранение без "дырок"
- Быстрый обход соседей
- Информация о рёбрах отдельно не сохраняется, ребра определяются через вершины
- Дополнительно сохраняем отображение вершин на индексы (для удобства поиска) 

---

### 2.7 Списки смежности ([GraphAdjList.h](../headers/graph/GraphAdjList.h))

**Особенности:**
- Для реализации использутеся библиотечная структура `std::list`
- Информация о рёбрах отдельно не сохраняется, ребра определяются через вершины
- Дополнительно сохраняем отображение вершин на индексы (для удобства поиска) 

---

### 2.8 Структура с оглавлением ([GraphStructured.h](../headers/graph/GraphStructured.h))

**Особенности:**
- Для хранинения "оглавления" используется стандартный контейнер `std::map` с ключом-вершиной и парой <итератор на первый элемент списка ребер для ключа, длина списка> в качестве значения
- Информация о рёбрах отдельно не сохраняется, ребра определяются через вершины

---

### 2.9 Список вершин и список рёбер ([GraphVertexEdgeList.h](../headers/graph/GraphVertexEdgeList.h))

**Особенности:**
- При реализации использовались стандартные контейнеры `std::list`, абстрагирующие "сырые" указатели 
- Сохраняется информация о ребрах
- Дополнительно сохраняется индекс вершин для удобства поиска
