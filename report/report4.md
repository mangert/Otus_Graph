## Реализованные алгоритмы

Все алгоритмы реализованы в файле [`graph_algorithms_shortest_path.h`](../headers/algorithms/graph_algorithms_shortest_path.h) и используют единый интерфейс `IGraph`. Это позволяет алгоритмам работать с любым представлением графа (матрица смежности, матрица инцидентности, список ребер и т.д.) без изменения кода.

Для хранения промежуточных данных используются:
- [`PriorityQueue`](../headers/data_structures/PriorityQueue.h) — собственная реализация бинарной кучи (min-heap) для алгоритма Дейкстры
- Общая структура [`Edge`](../headers/algorithms/graph_algorithms_common.h) для представления ребер

Оба алгоритма возвращают дерево кратчайших путей в виде массива ребер. Алгоритмы работают как с **ориентированными**, так и с **неориентированными** графами. Вес ребра (`EdgeInfo`) должен конвертироваться в числовой тип (`Distance`), используемый для хранения расстояний.

---

### 1. Алгоритм Дейкстры (Dijkstra)

**Функция:** [`dijkstra`](../headers/algorithms/graph_algorithms_shortest_path.h#L39-dijkstra)

Алгоритм Дейкстры находит кратчайшие пути от стартовой вершины до всех остальных вершин во взвешенном графе с **неотрицательными весами** ребер.

**Сложность:** O((V + E) log V) с использованием приоритетной очереди

**Возвращаемое значение:**
- `std::optional<std::vector<Edge<Vertex>>>` — вектор ребер, составляющих дерево кратчайших путей
- `std::nullopt` — в следующих случаях:
  - Граф пуст (нет вершин)
  - Стартовая вершина не принадлежит графу
  - Граф содержит ребра с отрицательным весом
  - Граф несвязный (не все вершины достижимы из стартовой)

**Особенности реализации:**
- Использует собственную реализацию `PriorityQueue` (бинарная куча) для эффективного извлечения вершины с минимальным расстоянием
- Поддерживает отложенное обновление: в очереди могут находиться устаревшие записи, которые отсеиваются проверкой `visited[current_idx] || current.distance > distances[current_idx]`
- Работает как с ориентированными, так и с неориентированными графами

### 2. Алгоритм Беллмана-Форда (Bellman-Ford)

**Функция:** [`bellmanFord`](../headers/algorithms/graph_algorithms_shortest_path.h#L160-bellmanFord)

Алгоритм Беллмана-Форда находит кратчайшие пути от стартовой вершины до всех остальных вершин во взвешенном графе. В отличие от Дейкстры, **поддерживает отрицательные веса** ребер, но не работает при наличии отрицательных циклов.

**Сложность:** O(V × E)

**Возвращаемое значение:**
- `std::optional<std::vector<Edge<Vertex>>>` — вектор ребер, составляющих дерево кратчайших путей
- `std::nullopt` — в следующих случаях:
  - Граф пуст (нет вершин)
  - Стартовая вершина не принадлежит графу
  - Граф содержит отрицательный цикл, достижимый из стартовой вершины
  - Граф несвязный (не все вершины достижимы из стартовой)

**Особенности реализации:**
- Выполняет V-1 проходов релаксации всех ребер
- После основного цикла выполняет дополнительный проход для обнаружения отрицательных циклов
- Для неориентированных графов каждое ребро добавляется в обоих направлениях

### Пользовательская структура ColourWay

Для проверки гибкости алгоритмов была создана пользовательская структура:

```cpp
struct ColourWay {
    double dist;
    std::string colour;
    
    operator double() const {
        if (colour == "red") return dist * 2;
        if (colour == "green") return dist * 0.5;
        return dist;
    }
};
```

Алгоритмы успешно преобразуют `ColourWay` в `double` через оператор приведения, структура условная для демонстрации

## Заключение

В рамках домашнего задания реализованы два алгоритма поиска кратчайших путей:

1. **Алгоритм Дейкстры** — эффективный алгоритм для графов с неотрицательными весами (O((V+E) log V))
2. **Алгоритм Беллмана-Форда** — универсальный алгоритм, работающий с отрицательными весами и обнаруживающий отрицательные циклы (O(V×E))

Оба алгоритма:
- Работают с любым представлением графа, реализующим интерфейс `IGraph`
- Поддерживают как ориентированные, так и неориентированные графы
- Используют общую структуру `Edge` для представления результата
- Возвращают дерево кратчайших путей в виде массива ребер
- Корректно обрабатывают краевые случаи (пустой граф, неверная стартовая вершина, несвязный граф)
- Поддерживают пользовательские типы весов через операторы преобразования

Тестирование подтверждает корректную работу алгоритмов на всех рассмотренных графах, включая:
- Графы с неотрицательными весами (оба алгоритма)
- Графы с отрицательными весами (только Беллман-Форд)
- Ориентированные и неориентированные графы
- Пользовательские структуры с операторами приведения типов
