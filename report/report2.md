## Реализованные алгоритмы

Все алгоритмы реализованы в файле [`graph_algorithms.h`](../headers/algorithms/graph_algorithms.h) и используют единый интерфейс `IGraph`. Это позволяет алгоритмам работать с любым представлением графа (матрица смежности, списки смежности, векторы смежности и т.д.) без изменения кода.

Для хранения промежуточных данных используются собственные структуры данных, реализованные на базе класса [`List`](../headers/data_structures/List.h) из предыдущих домашних заданий:
- [`Stack`](../headers/data_structures/Stack.h) — для обхода в глубину и хранения результатов
- [`Queue`](../headers/data_structures/Queue.h) — для алгоритма Демукрона

---

### 1. Алгоритм Демукрона (ярусная топологическая сортировка)

**Функция:** [`demukron`](../headers/algorithms/graph_algorithms_topological.h#L-demukron)

Алгоритм выполняет ярусную топологическую сортировку ориентированного ациклического графа (DAG). На каждом шаге выделяются все вершины с нулевой полустепенью захода, которые образуют очередной уровень. Если на каком-то шаге таких вершин нет, но остались необработанные вершины — граф содержит цикл.

**Сложность:** O(V + E)

**Возвращаемое значение:**
- `std::optional<std::vector<std::vector<Vertex>>>` — вектор уровней (каждый уровень — вектор вершин)
- `std::nullopt` — если граф содержит цикл

---

### 2. Алгоритм Тарьяна (топологическая сортировка)

**Функция:** [`tarjan`](../headers/algorithms/graph_algorithms_topological.h#L-tarjan)

Классический алгоритм Тарьяна для топологической сортировки. Использует рекурсивный обход в глубину с отслеживанием состояния вершин:
- `none` — вершина не посещена
- `seen` — вершина в процессе обхода (в стеке рекурсии)
- `cplt` — вершина полностью обработана

Если при обходе встречается вершина в состоянии `seen` — обнаружен цикл.

**Сложность:** O(V + E)

**Возвращаемое значение:**
- `std::optional<std::vector<Vertex>>` — топологический порядок вершин
- `std::nullopt` — если граф содержит цикл

---

### 3. Алгоритм поиска мостов

**Функция:** [`findBridges`](../headers/algorithms/graph_algorithms_topological.h#L-findBridges)

Алгоритм Тарьяна для поиска мостов в неориентированном графе. Использует обход в глубину с вычислением `lowlink` для каждой вершины. Ребро `(u, v)` является мостом, если `lowlink[v] > index[u]`, то есть из поддерева `v` нет обратных ребер, достигающих `u` или выше.

**Сложность:** O(V + E)

**Возвращаемое значение:**
- `std::vector<std::pair<Vertex, Vertex>>` — список ребер-мостов

### Тестовые графы

#### Для Демукрона и Тарьяна (ориентированные графы)

| Название | Описание | Результат |

| Двусвязный граф | Квадрат с диагональю | Мостов нет |
| Граф с мостами | Два треугольника, соединенные мостом | Один мост (2,3) |

## Заключение

В рамках домашнего задания реализованы три алгоритма на графах:

1. **Алгоритм Демукрона** — ярусная топологическая сортировка
2. **Алгоритм Тарьяна** — топологическая сортировка с обнаружением циклов
3. **Алгоритм поиска мостов** — нахождение критических ребер в неориентированном графе

Все алгоритмы:
- Работают с любым представлением графа, соответствующего интерфейсу `IGraph`, при этом для алгоритмов Демукрона и Тарьяна установлено допускаются только ориентированные графы, для поиска мостов - только неориентированные
- Используют собственные структуры данных (`Stack`, `Queue`) на базе `List` из предыдущих ДЗ
- Протестированы на наборе графов, включая ациклические, циклические, деревья, циклы и полные графы
- Корректно обрабатывают графы с циклами (возвращают `std::nullopt` для Демукрона и Тарьяна)
