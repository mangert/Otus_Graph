# Отчет по домашнему заданию: Алгоритмы построения минимального остовного дерева (MST)

---

## Реализованные алгоритмы

Все алгоритмы реализованы в файле [`graph_algorithms_mst.h`](../headers/algorithms/graph_algorithms_mst.h) и используют единый интерфейс `IGraph`. Это позволяет алгоритмам работать с любым представлением графа (матрица смежности, матрица инцидентности, список ребер и т.д.) без изменения кода.

Для хранения промежуточных данных используются:
- [`UnionFind`](../headers/data_structures/UnionFind.h) — собственная реализация системы непересекающихся множеств с произвольным типом вершин, сжатием пути и объединением по рангу
- [`PriorityQueue`](../headers/data_structures/PriorityQueue.h) — собственная реализация бинарной кучи (min-heap) для алгоритма Прима

Все три алгоритма работают только с **неориентированными взвешенными графами**. Вес ребра (`EdgeInfo`) должен быть сравнимым (`Comparable`), что позволяет сортировать ребра и находить минимальные.

---

### 1. Алгоритм Краскала (Kruskal)

**Функция:** [`kruskal`](../headers/algorithms/graph_algorithms_mst.h#L-kruskal)

Алгоритм Краскала строит минимальное остовное дерево, последовательно добавляя ребра в порядке возрастания веса, если они соединяют разные компоненты связности.

**Сложность:** O(E log E) из-за сортировки ребер

**Возвращаемое значение:**
- `std::optional<std::vector<Edge<Vertex>>>` — вектор ребер, составляющих MST
- `std::nullopt` — если граф несвязный

**Особенности реализации:**
- Использует `UnionFind` для эффективного отслеживания компонент связности
- Для неориентированных графов каждое ребро в `getEdges()` возвращается один раз

---

### 2. Алгоритм Прима (Prim)

**Функция:** [`prim`](../headers/algorithms/graph_algorithms_mst.h#L-prim)

Алгоритм Прима строит MST, начиная с произвольной стартовой вершины и на каждом шаге добавляя ближайшую вершину, не входящую в дерево.

**Сложность:** O(E log V) с использованием приоритетной очереди

**Возвращаемое значение:**
- `std::optional<std::vector<Edge<Vertex>>>` — вектор ребер, составляющих MST
- `std::nullopt` — если граф несвязный

**Особенности реализации:**
- Использует собственную реализацию `PriorityQueue` (бинарная куча) для эффективного извлечения минимального элемента
- Поддерживает отложенное обновление: в очереди могут находиться устаревшие записи, которые отсеиваются проверкой `in_mst`
- Начинается с произвольной стартовой вершины (передается как параметр)

---

### 3. Алгоритм Борувки (Borůvka)

**Функция:** [`boruvka`](../headers/algorithms/graph_algorithms_mst.h#L-boruvka)

Алгоритм Борувки для построения MST. На каждом шаге для каждой компоненты связности выбирается самое дешевое исходящее ребро, и все выбранные ребра добавляются в MST.

**Сложность:** O(E log V) (на каждом шаге просматриваются все ребра)

**Возвращаемое значение:**
- `std::optional<std::vector<Edge<Vertex>>>` — вектор ребер, составляющих MST
- `std::nullopt` — если граф несвязный

**Особенности реализации:**
- Использует `UnionFind` для отслеживания компонент
- На каждом шаге перебираются все ребра
- Для каждой компоненты хранится самое дешевое исходящее ребро
- Параллельно обрабатываются все компоненты

## Структуры данных

### UnionFind

**Файл:** [`UnionFind.h`](../headers/data_structures/UnionFind.h)

Собственная реализация системы непересекающихся множеств, поддерживающая произвольный тип вершин.

**Основные методы:**
- `makeSet(const T& data)` — создание нового множества с элементом
- `root(const T& data)` — поиск корня множества (с сжатием пути)
- `isConnected(const T& x, const T& y)` — проверка принадлежности к одному множеству
- `unite(const T& x, const T& y)` — объединение множеств (по рангу)

**Особенности:**
- Использует `std::map` для отображения произвольного типа вершин на индексы
- Реализовано объединение по рангу (union by rank) для балансировки дерева

### PriorityQueue

**Файл:** [`PriorityQueue.h`](../headers/data_structures/PriorityQueue.h)

Собственная реализация бинарной кучи (min-heap) для алгоритма Прима.

**Основные методы:**
- `push(const T& value)` — добавление элемента
- `push(T&& value)` — добавление элемента с перемещением
- `pop()` — удаление минимального элемента
- `top()` — доступ к минимальному элементу
- `empty()` — проверка на пустоту
- `size()` — количество элементов

**Особенности:**
- Реализована на динамическом массиве (`std::vector`)
- Поддерживает произвольный компаратор (по умолчанию `std::less` для min-heap)
- Использует `[[no_unique_address]]` для оптимизации хранения stateless компаратора
- Ограничения: компаратор должен быть вызываем с двумя элементами и возвращать `bool`

## Заключение

В рамках домашнего задания реализованы три алгоритма построения минимального остовного дерева (MST):

1. **Алгоритм Краскала** — сортировка ребер с использованием Union-Find
2. **Алгоритм Прима** — жадный алгоритм с приоритетной очередью
3. **Алгоритм Борувки** — параллельный алгоритм, на каждом шаге добавляющий лучшие ребра для каждой компоненты

Все алгоритмы:
- Работают с любым представлением графа, реализующим интерфейс `IGraph`
- Используют собственные структуры данных (`UnionFind`, `PriorityQueue`)
- Корректно обрабатывают несвязные графы (возвращают `std::nullopt`)
- Протестированы на наборе графов, включая деревья, циклы, полные графы, графы с одинаковыми весами и краевые случаи (одна вершина)

Тестирование подтверждает, что все три алгоритма возвращают корректное минимальное остовное дерево (или `nullopt` для несвязных графов) для всех рассмотренных случаев. Небольшие различия в порядке ребер в результате связаны с особенностями реализации и не влияют на корректность, так как графы неориентированные.
