# Graph Representations

Учебный проект по реализации различных способов представления графов и алгоритмов на графах на C++.

## Состав проекта

### Заголовочные файлы (`headers/`)
- `IGraph.h` - базовый интерфейс для всех представлений
- `graph_factory.h` - функции для создания графов из списков вершин и рёбер

### Реализации представлений (`headers/graph/`)
1. `GraphEnumeration.h` - перечисление множеств
2. `GraphAdjMatrix.h` - матрица смежности
3. `GraphIncMatrix.h` - матрица инцидентности
4. `GraphEdgeList.h` - перечень рёбер
5. `GraphAdjVectors.h` - векторы смежности
6. `GraphAdjArray.h` - массивы смежности
7. `GraphAdjList.h` - списки смежности
8. `GraphStructured.h` - структура с оглавлением
9. `GraphVertexEdgeList.h` - список вершин и список рёбер

### Структуры данных (`headers/data_structures/`)
- `List.h` - базовый класс списка
- `Stack.h` - стек на базе List
- `Queue.h` - очередь на базе List
- `UnionFind.h` - система непересекающихся множеств (Union-Find) с поддержкой произвольного типа вершин
- `PriorityQueue.h` - бинарная куча (приоритетная очередь) для алгоритмов MST и Дейкстры

### Алгоритмы (`headers/algorithms/`)
- `graph_algorithms_common.h` - общие структуры (`Edge`) для всех алгоритмов

- `graph_algorithms_topological.h` - алгоритмы топологической сортировки:
  - **Демукрона** - ярусная топологическая сортировка
  - **Тарьяна** - топологическая сортировка с обнаружением циклов
  - **Поиск мостов** - нахождение критических ребер в неориентированном графе

- `graph_algorithms_mst.h` - алгоритмы построения минимального остовного дерева (MST):
  - **Краскала (Kruskal)** - сортировка рёбер с использованием Union-Find
  - **Прима (Prim)** - жадный алгоритм с приоритетной очередью
  - **Борувки (Borůvka)** - параллельный алгоритм, добавляющий лучшие рёбра для каждой компоненты

- `graph_algorithms_shortest_path.h` - алгоритмы поиска кратчайших путей:
  - **Дейкстры (Dijkstra)** - эффективный алгоритм для графов с неотрицательными весами (O((V+E) log V))
  - **Беллмана-Форда (Bellman-Ford)** - универсальный алгоритм, работающий с отрицательными весами и обнаруживающий отрицательные циклы (O(V×E))

### Исходные файлы (`src/`)
- `main.cpp` - точка входа
- `tests/test_algorithms.cpp` - тестирование топологических алгоритмов
- `tests/test_mst.cpp` - тестирование MST алгоритмов
- `tests/test_shortest_path.cpp` - тестирование алгоритмов поиска кратчайших путей

### Отчёт (`report/`)
- `output.txt`  - результаты тестирования корректности представлений
- `output2.txt` - результаты тестирования топологических алгоритмов
- `output3.txt` - результаты тестирования MST алгоритмов
- `output4.txt` - результаты тестирования алгоритмов поиска кратчайших путей
- `report.md`...`report4.md` - подробные отчёты о выполненной работе (топологические алгоритмы, MST)

## Особенности

- Шаблонная архитектура с поддержкой произвольных типов вершин и информации о рёбрах
- Параметр `Directed` для выбора типа графа (ориентированный/неориентированный)
- Индексные структуры для быстрого поиска вершин и рёбер
- Полное покрытие тестами всех 9 представлений
- Алгоритмы работают с любым представлением через единый интерфейс `IGraph`
- Использование собственных структур данных (стек, очередь, Union-Find, приоритетная очередь) из предыдущих ДЗ

### Алгоритмы топологической сортировки
- Корректно обрабатывают графы с циклами (возвращают `std::nullopt`)
- Демукрон возвращает ярусное разбиение, Тарьян - линейный порядок

### Алгоритмы MST
- Все три алгоритма корректно обрабатывают несвязные графы (возвращают `std::nullopt`)
- Поддерживают произвольные сравнимые типы весов (`Comparable` концепт)
- Возвращают дерево в виде массива рёбер

### Алгоритмы поиска кратчайших путей
- **Дейкстра:** работает с неотрицательными весами, сложность O((V+E) log V)
- **Беллман-Форд:** работает с отрицательными весами, обнаруживает отрицательные циклы, сложность O(V×E)
- Оба алгоритма поддерживают ориентированные и неориентированные графы
- Возвращают дерево кратчайших путей в виде массива рёбер
- Поддерживают пользовательские типы весов через операторы преобразования в числовые типы
- Корректно обрабатывают краевые случаи: пустой граф, неверная стартовая вершина, несвязный граф
